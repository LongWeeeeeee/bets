# Ingame — Dota 2 Analytics Platform


> **Единый источник правды для всех ИИ-агентов.** Этот файл (`AGENTS.md`) — канонический.
> `CLAUDE.md`, `GEMINI.md`, `.cursorrules` и `.github/copilot-instructions.md` — симлинки на него.
> Правь ТОЛЬКО `AGENTS.md`. Claude Code, Codex, Gemini CLI, Cursor, Copilot читают одно и то же.


Система анализа и прогнозирования Dota 2 матчей: live-данные, букмекерские кэфы, ML-модели и ELO-рейтинги, связка с Telegram-ботом. Автор: alex.

⏺ Claude Code 会话管理：


  后台化已运行的会话
  
  没办法把已经运行中的 Claude Code 进程移到 nohup。nohup 必须在启动时使用。已开启的会话：
  
  - Ctrl+Z → bg：会让进程在后台暂停，不会工作。Claude Code 是交互式 TUI，需要终端 I/O。后台暂停后无法输入/看到输出。
  - Ctrl+C：会中断并退出会话，但会话历史会保留。下次 claude 启动可以 --continue 恢复最近的会话。
  
  多会话方案
  
  ┌──────────────────────────┬───────────────────────────────────────────┬────────────────────────────────────────────────┐
  │           方案           │                   命令                    │                    适用场景                    │
  ├──────────────────────────┼───────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ tmux/screen              │ tmux new -s claude1, 然后 Ctrl+B D 分离   │ ✅ 最佳 — 多个独立 session，随时 attach/detach │
  ├──────────────────────────┼───────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ nohup 启动新会话         │ nohup claude -p "задача" > out.log 2>&1 & │ 非交互模式 (-p/--print)，输出到文件            │

  最佳实践


  # 1. 创建 tmux session（推荐）
  tmux new -s ingame
  claude    # 在里面正常使用
  # Ctrl+B, D — detach，进程继续跑


  # 2. 回到这个 session
  tmux attach -t ingame


  # 3. 同时开第二个独立 session
  tmux new -s other-task
  claude --continue    # 恢复之前的会话，或开新会话


  总结


  - 当前会话：放不到后台。只能继续用，或 Ctrl+C 退出后用 --continue 恢复。
  - 未来会话：用 tmux。这才是正确的"把终端放后台"方案。
  - nohup 只适合非交互的 batch 模式 (claude -p "...")，不带 -p 的交互式 Claude Code 用 nohup 也会挂。
  - 没有内置守护进程功能 — Claude Code 是有状态的终端应用，不是 daemon。
