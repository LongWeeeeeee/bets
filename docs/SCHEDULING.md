# Schedule / Sleep Policy

Main-цикл (`base/cyberscore_try.py` `__main__`, ~27193) выбирает sleep по статусу `general()`. Цель цикла при активных матчах — `_POLL_CYCLE_TARGET_SECONDS` (env `CYBERSCORE_POLL_CYCLE_TARGET_SECONDS`, default = `SCHEDULE_NEAR_MATCH_POLL_SECONDS` = **30с**): после обработки спит остаток до цели.

## Env-параметры расписания (имя → default)

| env | default |
|---|---|
| `CYBERSCORE_SCHEDULE_POLL_SECONDS` | `30*60` (1800с) |
| `SCHEDULE_MAX_SLEEP_SECONDS` | `30*60` (cap сна) |
| `SCHEDULE_LONG_IDLE_THRESHOLD_SECONDS` | `30*60` |
| `SCHEDULE_WAKE_LEAD_SECONDS` | `30*60` (lead до старта матча) |
| `SCHEDULE_NEAR_MATCH_POLL_SECONDS` | `30` |
| `SCHEDULE_POST_START_POLL_SECONDS` | `3*60` (poll после старта, ожидание live) |
| `CYBERSCORE_POLL_CYCLE_TARGET_SECONDS` | `SCHEDULE_NEAR_MATCH_POLL_SECONDS` (30) |
| `CYBERSCORE_QUIET_HOURS_START_HOUR_MSK` | `0` |
| `CYBERSCORE_QUIET_HOURS_END_HOUR_MSK` | `6` |
| `CYBERSCORE_QUIET_HOURS_PROBE_ENABLED` | `1` |
| `CYBERSCORE_DELAYED_STALE_REFRESH_SECONDS` | `90` |

## Статусы → sleep

| Статус `general()` | Поведение |
|---|---|
| `None` / прочее | sleep = `max(0.5, target - elapsed)` (~30с цикл) |
| `series_finished` | fetch расписания → sleep до ближайшего матча или cap `SCHEDULE_MAX_SLEEP_SECONDS` |
| `__sleep_until_schedule__` | sleep ровно до ближайшего матча (`NEXT_SCHEDULE_SLEEP_SECONDS`) |
| `__sleep_wait_for_live_after_schedule__` | `SCHEDULE_POST_START_POLL_SECONDS` (3 мин) — матч стартовал, ждём live |
| `__sleep_cyberscore_quiet_hours__` | sleep до конца ночного окна (00:00–06:00 MSK) |

## Quiet hours (00:00–06:00 MSK)
В режиме `DLTV_SOURCE_MODE=cyberscore` в начале `general()` проверяется `_cyberscore_quiet_hours_sleep_seconds_with_probe()`. Если в окне — bookmaker prefetch воркер останавливается, runtime спит до конца окна (probe может разбудить раньше при появлении матча, если `CYBERSCORE_QUIET_HOURS_PROBE_ENABLED=1`).

## Заметки
- Все sleep — через `_sleep_interruptible(...)` (прерывается admin-командами/событиями).
- Цикл ~30с при активных матчах — норма; основное время уходит на Camoufox page loads (карточки + individual match pages) и bookmaker prefetch.
